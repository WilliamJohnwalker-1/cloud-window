# .omo Execution Preferences

## 1) 计划阶段（默认省 token）

- 建立计划时，默认**不执行双端完整手工真实 QA**。
- 默认输出改为：
  - 关键风险点
  - 最小可执行测试清单（移动端 / Web）
  - 预期结果与失败判定
- 仅在以下情况执行大规模手工 QA：
  - 用户明确要求代跑
  - 发布阻断风险且无法通过静态检查/单测覆盖
- 其他高开销动作同样遵循“最小必要”原则：
  - 避免冗长日志转储
  - 避免无差别大范围并行检索

## 2) 执行计划阶段（账号与数据保护）

- **指定联调用测试账号（Web）**：`testadmin@example.com`
- **禁止**自行注册临时账号或创建临时业务账号（主代理/子代理均适用）。
- **禁止**子代理在任何环境创建账号、写入 `auth.users`、写入 `profiles` 测试用户。
- 需要登录凭据时，必须先向用户请求测试账号/角色；仅允许使用用户明确提供的现有账号。
- 严禁执行任何“注册测试号/提权测试号/补档案到生产库”的脚本。
- 目标：避免污染 Supabase 数据库，保持数据可追溯与环境干净。
